# نظام إدارة متجر التخرج (Graduation Store Management System)

نظام متكامل، حديث، واحترافي مصمم خصيصاً لإدارة مبيعات وإيجارات مستلزمات التخرج (قبعات، كابات، أوشحة، بروشات، وإكسسوارات) بالكامل باللغة العربية مع واجهة تدعم اتجاه الكتابة من اليمين إلى اليسار (RTL).

النظام مبني باستخدام تقنيات حديثة ومفتوحة المصدر لضمان السرعة، الأمان، والتوافقية الكاملة.

---

## 🛠️ التقنيات المستخدمة (Tech Stack)

*   **الواجهة الأمامية (Frontend)**: React, TypeScript, Vite, Tailwind CSS (خطوط Cairo و Tajawal).
*   **الخلفية وقاعدة البيانات (Backend & Database)**: Node.js, Express, Prisma ORM, SQLite (مع إمكانية التبديل إلى PostgreSQL أو Supabase بثلاث خطوات).
*   **المصادقة والأمان (Authentication & Security)**: JWT (JSON Web Tokens), Hashing Passwords (bcryptjs), Zod Validation.
*   **تصدير التقارير (Reports Export)**: تصدير إلى Excel و CSV (عبر xlsx) وتصدير التقارير كملف PDF (عبر jsPDF).

---

## 🚀 طريقة تشغيل وتثبيت النظام محلياً

يرجى اتباع الخطوات التالية لتشغيل النظام على جهازك المحلي:

### 1. تثبيت الحزم البرمجية (Dependencies)
افتح موجه الأوامر (Terminal) في المجلد الرئيسي للمشروع، ثم نفذ الأمر التالي لتثبيت حزم المجلد الرئيسي والـ Backend والـ Frontend بضغطة واحدة:
```bash
npm run install:all
```

### 2. تهيئة قاعدة البيانات وتغذية البيانات التجريبية (Database & Seed)
قم بتشغيل الهجرة لإنشاء الجداول في قاعدة بيانات SQLite المحلية وتغذية مستخدمين تجريبيين وحركات مالية لتجميل لوحة الإحصائيات العامة:
```bash
npm run db:setup
```

### 3. تشغيل النظام في بيئة التطوير (Run Development Server)
قم بتشغيل الخادمين معاً (خادم Express على منفذ 5000 وخادم React-Vite على منفذ 5173 مع التوجيه التلقائي للمصادقة):
```bash
npm run dev
```
بعد تشغيل الأمر، افتح متصفحك على الرابط: **`http://localhost:5173`**

## 🔌 التبديل من SQLite إلى PostgreSQL أو Supabase
تم تصميم النظام ليكون جاهزاً للعمل التجاري المباشر. إذا أردت التبديل لقاعدة بيانات PostgreSQL أو Supabase في بيئة الإنتاج:

1.  افتح ملف `backend/prisma/schema.prisma` وقم بتغيير سطر الـ `provider` من `"sqlite"` إلى `"postgresql"`:
    ```prisma
    datasource db {
      provider = "postgresql"
      url      = env("DATABASE_URL")
    }
    ```
2.  افتح ملف المتغيرات البيئية `backend/.env` وقم بتحديث `DATABASE_URL` برابط الاتصال الخاص بقاعدة بيانات PostgreSQL:
    ```env
    DATABASE_URL="postgresql://username:password@localhost:5432/db_name?schema=public"
    ```
3.  قم بإعادة توليد الهجرة وتغذية قاعدة البيانات بالأمر التالي:
    ```bash
    cd backend
    npx prisma migrate dev --name init
    npx prisma db seed
    ```

## 📄 تصنيفات وأنواع المنتجات المتاحة
يدعم النظام خيار **"أخرى" (Other)** في كافة القوائم المنسدلة في الفاتورة، وعند اختياره يظهر حقل نصي إضافي فوراً لكتابة أي قيمة مخصصة:
*   **الكاب (Graduation Cap)**: كاب الفراشة / فراشة طويل، كاب TLC (القياسات S, 1, 2, 3, أخرى)، كاب أمريكي، كاب كويتي (الألوان: فضي، ذهبي، سادة، أخرى)، أو أخرى.
*   **الطاقية (Graduation Hat)**:
    *   *حالة البيع*: سادة، تطريز، طباعة، أخرى.
    *   *حالة الإيجار*: تسجل كمية، قيمة الإيجار، وقيمة التأمين المستلم، وتاريخ الإرجاع المتوقع.
*   **الوشاح (Graduation Sash)**:
    *   *حالة البيع*: سادة، تطريز، طباعة، وتاريخ التخرج وتاريخ التسليم.
    *   *حالة الإيجار*: تسجل كمية، قيمة إيجار، تأمين، تاريخ الإرجاع المتوقع وتاريخ التخرج.
*   **البروش (Graduation Brooch)**: بروش سنة التخرج، بروش بالاسم، أو أخرى.
*   **الإكسسوارات (Graduation Accessories)**: مسبحة تخرج، شارة، أو أي إكسسوار مخصص باسمه.

## 🖨️ نظام طباعة المستندات (Print Layouts)
يدعم النظام طباعة حرارية أو ورقية منسقة وخالية من أشرطة التنقل لكل من:
1.  **الفاتورة التجارية (Invoice)**: تفاصيل البيع والإيجار والمتبقي والخصم للزبون.
2.  **عقد وإيصال الإيجار (Rental Receipt)**: يحتوي على شروط والتزامات المستأجر ومبلغ التأمين المسترد وتوقيعات الاستلام.
3.  **إيصال المقبوضات المالي (Payment Receipt)**: يسجل القيمة المقبوضة وطريقتها نقداً أو تحويل أو بطاقة مع رصيد العميل المتبقي.
4.  **إيصال المرتجع (Return Receipt)**: يوثق عدد القطع المرجعة وحالتها وفحصها عند التسليم.
